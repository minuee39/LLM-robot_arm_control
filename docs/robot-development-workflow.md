# Isaac Sim · ROS 2 · YOLO 개발 기록 및 주간 요약 워크플로

## 목적

Isaac Sim 시뮬레이션, ROS 2/MoveIt 2 제어, YOLO 비전 인식 작업을 동일한 형식으로 반복 기록하고, 변경 사항과 미해결 문제를 주기적으로 요약한다.

이 문서는 워크플로와 자동화 **초안**만 정의한다. 외부 전송, Git 커밋·푸시, 이슈 생성, 일정 변경은 사용자 승인 없이 실행하지 않는다.

## 권장 디렉터리

```text
robot-project/
├── README.md
├── logs/
│   ├── daily/
│   └── weekly/
├── decisions/
├── issues/
├── configs/
└── experiments/
```

- `logs/daily`: 작업 세션별 원본 기록
- `logs/weekly`: 주간 변경 요약
- `decisions`: 좌표계, 토픽, 모델, 플래너 등 장기 결정
- `issues`: 떨림, 불필요한 관절 회전, TF 오류, 인식 실패
- `configs`: 실행에 사용한 설정의 추적 가능한 사본
- `experiments`: YOLO 모델별 FPS, 좌표 오차, 제어 파라미터 비교

## 기록 흐름

1. 작업 시작 전에 목표와 현재 환경을 기록한다.
2. 코드·설정 변경마다 변경 이유와 관련 파일을 남긴다.
3. 실행 명령, 입력 조건, 출력 결과를 함께 저장해 재현 가능하게 만든다.
4. 성능 수치는 측정 방법과 단위를 함께 기록한다.
5. 실패도 삭제하지 않고 원인, 시도, 결과를 기록한다.
6. 실제 로봇 적용 전에는 속도·가속도 제한, 충돌 검사, 비상정지 상태를 확인한다.

## 주간 요약 자동화 초안

권장 실행 시각은 매주 일요일 21:00(KST)이다.

```yaml
name: 로봇 프로젝트 주간 변경 요약
schedule:
  frequency: weekly
  weekday: sunday
  time: "21:00"
  timezone: Asia/Seoul

scope:
  include:
    - logs/daily
    - decisions
    - issues
    - configs
    - source-code-diff
  exclude:
    - API 키
    - 인증 토큰
    - 개인정보
    - 대용량 데이터셋
    - 모델 가중치

output:
  path: logs/weekly/YYYY-Www-summary.md
  external_delivery: disabled
  require_approval_before:
    - 외부 저장소 업로드
    - 메시지 또는 이메일 전송
    - Git commit 또는 push
    - GitHub 이슈 생성
    - 일정 변경
```

## 주간 요약 프롬프트

```text
지난 7일간의 Isaac Sim, ROS 2, MoveIt 2, YOLO 작업 기록과
승인된 코드 변경 내역을 분석해 한국어 주간 보고서를 작성한다.

다음 항목을 포함한다.
1. 이번 주 목표와 완료 사항
2. 구성요소별 변경 사항
3. 전체 데이터 흐름에 미친 영향
4. 검증 결과와 측정값
5. 새로 발생한 오류와 해결 여부
6. 아직 재현되지 않았거나 근거가 부족한 추정
7. 결정된 사항과 변경 이유
8. 다음 주 우선순위 최대 5개
9. 실제 로봇 적용 전 안전 확인 사항

사실, 추정, 제안을 명확히 구분한다.
기록에 없는 성능 수치나 성공 결과를 생성하지 않는다.
API 키, 토큰, 개인정보는 출력하지 않는다.
외부 시스템에 기록·전송·커밋하지 않는다.
외부 변경이 필요하면 실행하지 말고 '승인 필요 작업'에만 정리한다.
```

## 실패 시 동작

- 기존 주간 보고서를 덮어쓰지 않는다.
- 입력 기록이 없으면 빈 성공 보고서를 만들지 않는다.
- 분석 실패 시 누락 파일과 실패 원인만 별도 실패 기록에 남긴다.
- 외부 전송 또는 저장소 변경에 실패해도 자동 재시도하지 않고 승인 대기 항목으로 남긴다.
- 측정 근거가 없는 값은 `미측정`으로 표시한다.

## 보안 및 프라이버시

- `.env`, 토큰, 인증서, 카메라 개인정보 영상은 요약 대상에서 제외한다.
- 명령 출력에 포함된 인증정보는 기록 전에 제거한다.
- 실제 로봇 명령은 자동 요약 과정에서 실행하지 않는다.
- 외부 시스템 변경은 항상 사용자의 명시적 승인을 받은 뒤 수행한다.
